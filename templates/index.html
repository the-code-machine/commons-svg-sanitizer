Validate SVG files before uploading to Commons

MediaWiki strips disallowed elements, attributes, and unsafe URLs from every SVG uploaded to Wikimedia Commons. This tool reports what would be removed and gives you a clean copy you can upload directly. Files are processed in memory and never stored on the server.

    How to use

    1. Drop or choose an .svg file (or load a sample above to try the tool).
    2. Review the Findings list. Items marked Blocker will be stripped or rejected by Commons. Warning items are usually editor-specific elements (e.g. Inkscape) that won't render.
    3. Use the side-by-side preview to confirm the cleaned version still looks correct.
    4. Click Download and upload the cleaned file to Special:Upload.

    Commons SVG guidelines

    Summarised from Help:SVG, Help:SVG guidelines, and Commons:Transition to SVG. Refer to the originals for the full text.

    Root must be <svg>

    The root element must be <svg>. Standalone XML other than SVG is rejected.

    No scripts or interactive content

    <script>, <foreignObject>, <iframe>, and event handlers (onclick, onload, …) are stripped during sanitisation.

    URLs must be safe

    href values may only point to #, https://, or http://. javascript: and other schemes are rejected.

    Allowed namespaces only

    MediaWiki accepts SVG, Inkscape, Adobe, Creative Commons, and W3C RDF namespaces. XHTML inside an SVG is rejected.

    Use librsvg-friendly fonts

    Commons renders SVGs with librsvg, which only has a limited font list. Prefer 'Liberation Sans', Arial, sans-serif and avoid Inkscape "flowed text".

    Validate before uploading

    A well-formed SVG renders consistently. Beyond this tool you can also use the W3C Markup Validator.
